
\section{EventBusClient}\label{sec:eventbusclient}

\subsection{Overview}\label{subsec:overview}
The \textbf{EventBusClient} provides a high-level API for interacting with a RabbitMQ-based event bus system.
It abstracts low-level AMQP details and supports dynamic loading of project-specific plugins for serializers, exchange handlers, and message structures.
This client is designed for modularity and ease of use across multiple projects.

---

\subsection{Features}\label{subsec:features}
\begin{itemize}
  \item \textbf{Dynamic plugin loading}: Load custom serializers, exchange handlers, and message classes at runtime from a configurable plugins directory.
  \item \textbf{Thread-safe publishing}: Supports safe publishing from multiple threads with minimal locking overhead.
  \item \textbf{Automatic reconnect}: Automatically reconnects to RabbitMQ after connection loss (if enabled in configuration).
  \item \textbf{Pluggable architecture}: Supports both built-in and project-specific extensions.
  \item \textbf{Supports JSON, Protobuf, Pickle serializers}.
\end{itemize}

\subsection{Architecture}\label{subsec:architecture}
The EventBusClient is built around a modular architecture that allows for easy extension and customization. It consists of the following components:
\begin{itemize}
  \item \textbf{Base Classes}: Define interfaces for exchange handlers, serializers, and messages.
  \item \textbf{Plugins Directory}: Contains custom implementations of the base classes.
  \item \textbf{Configuration File}: Specifies the plugins path, RabbitMQ connection details, and selected implementations.
  \item \textbf{EventBusClient Class}: The main class that interacts with RabbitMQ and manages plugins.
  \item \textbf{Message Classes}: Define the structure of messages sent and received over the event bus.
  \item \textbf{Exchange Handlers}: Handle the declaration and publishing of messages to RabbitMQ exchanges.
  \item \textbf{Serializers}: Convert messages to and from byte streams for transmission over RabbitMQ.
\end{itemize}
The client uses a combination of built-in plugins and user-defined plugins to provide flexibility in message handling and serialization.
\subsection{Installation}\label{subsec:installation}
To install the EventBusClient, use pip to install the package from PyPI:
\begin{verbatim}
pip install event-bus-client
\end{verbatim}
Ensure you have RabbitMQ server running and accessible at the specified host and port in the configuration file.
\subsection{Dependencies}\label{subsec:dependencies}

The EventBusClient requires the following dependencies:
\begin{itemize}
  \item \textbf{pika}: For RabbitMQ communication.
  \item \textbf{protobuf}: For Protocol Buffers serialization (if using ProtobufSerializer).
  \item \textbf{jsonpickle}: For JSON serialization (if using JSONSerializer).
  \item \textbf{pickle}: For Pickle serialization (if using PickleSerializer).
\end{itemize}
Ensure these dependencies are installed in your Python environment. You can install them using pip:
\begin{verbatim}
pip install pika protobuf jsonpickle
\end{verbatim}

\subsection{Configuration}\label{subsec:configuration}
The \texttt{config.jsonp} file controls all aspects of the EventBusClient, including plugin paths, RabbitMQ connection, and selected implementations.

\begin{verbatim}
{
  "plugins_path": "./plugins",
  "host": "localhost",
  "port": 5672,
  "serializer": "PickleSerializer",
  "exchange_handler": "XRTopicExchangeHandler",
  "message_class": "ListenerEventMsg",
  "threadsafe_publish": true,
  "auto_reconnect": true,
  "qos_prefetch": 10
}
\end{verbatim}

\subsection{Usage}\label{subsec:usage}

\subsubsection{Initialization}
Create an EventBusClient instance from configuration:
\begin{verbatim}[language=Python]
from event_bus.client import EventBusClient

client = await EventBusClient.from_config("./config/config.jsonp")
\end{verbatim}

---

\subsubsection{Publishing Messages}
Send a message using the configured serializer and exchange handler:
\begin{verbatim}[language=Python]
msg = ListenerEventMsg(event="start_cycle", timestamp=1234567890)
await client.send("zone1.topic.start", msg)
\end{verbatim}

Enable thread-safe publish from a non-async thread:
\begin{verbatim}[language=Python]
client.send("zone1.topic.alert", msg, threadsafe=True)
\end{verbatim}

---

\subsubsection{Subscribing to Messages}
Subscribe to a routing key and handle incoming messages:
\begin{verbatim}[language=Python]
async def on_message(msg: ListenerEventMsg):
    print(f"Received event: {msg.event} at {msg.timestamp}")

await client.on("zone1.#", ListenerEventMsg, on_message)
\end{verbatim}

\subsection{Example: Producer and Consumer}
This example (from \texttt{test.py}) starts a producer and a consumer in separate processes:
\begin{verbatim}[language=Python]
# Start consumer
consumer = Process(target=run_process, args=(consumer_process, config_path))
consumer.start()

# Start producer
producer = Process(target=run_process, args=(producer_process, config_path))
producer.start()
# Wait for both to finish
producer.join()
consumer.join()
\end{verbatim}

\subsection{Example: Custom Plugin Structure}\label{subsec:example:-custom-plugin-structure}
To create a custom plugin, follow these steps:
\begin{itemize}
  \item Create a directory for your plugin in the configured plugins path.
  \item Implement the required interfaces (e.g., BaseExchangeHandler, BaseMessage, BaseSerializer).
  \item Register your plugin in the \texttt{config.jsonp} file.
  \item Ensure your plugin is discoverable by the EventBusClient.
  \item Place your plugin code in the plugins directory, following the structure:
  \item For example, if your plugin is named \texttt{CustomPlugin}, the directory structure should look like this:
  \item \begin{verbatim}
    plugins/
    |-- CustomPlugin/
        |-- __init__.py
        |-- custom_exchange_handler.py
        |-- custom_message.py
        |-- custom_serializer.py
    \end{verbatim}
\end{itemize}
\subsection{Example: Built-in Plugins}
The EventBusClient comes with several built-in plugins that can be used directly or extended:
\begin{itemize}
  \item \textbf{XRTopicExchangeHandler}: Handles topic exchanges with RabbitMQ.
  \item \textbf{ListenerEventMsg}: A default message class for listener events.
  \item \textbf{PickleSerializer}: Serializes messages using Python's Pickle module.
  \item \textbf{JSONSerializer}: Serializes messages to JSON format.
  \item \textbf{ProtobufSerializer}: Serializes messages using Protocol Buffers.
\end{itemize}
These plugins can be used as-is or extended to create custom functionality.

\subsection{Example: Built-in Configuration}
The built-in configuration file \texttt{config.jsonp} provides a starting point for configuring the EventBusClient:
\begin{verbatim}
{
  "plugins_path": "./plugins",
  "host": "localhost",
  "port": 5672,
  "serializer": "PickleSerializer",
  "exchange_handler": "XRTopicExchangeHandler",
  "message_class": "ListenerEventMsg",
  "threadsafe_publish": true,
  "auto_reconnect": true,
  "qos_prefetch": 10
}
\end{verbatim}

This configuration specifies the plugins path, RabbitMQ connection details, serializer, exchange handler, message class, and other options.


\subsection{Example: Custom Exchange Handler}\label{subsec:example:-custom-exchange-handler}
To create a custom exchange handler, implement the required interface and place it in the plugins directory. For example, a custom exchange handler might look like this:
\begin{verbatim}[language=Python]
from event_bus.plugins import BaseExchangeHandler
class CustomExchangeHandler(BaseExchangeHandler):
    def declare_exchange(self, channel):
        # Custom exchange declaration logic
        pass

    def publish(self, channel, routing_key, message):
        # Custom publish logic
        pass
# Register the plugin in config.jsonp
{
  "plugins_path": "./plugins",
  "exchange_handler": "CustomExchangeHandler"
}
\end{verbatim}

\subsection{Example: Custom Message Class}\label{subsec:example:-custom-message-class}
To create a custom message class, define it in your project and register it in the configuration:
\begin{verbatim}[language=Python]
from event_bus.plugins import BaseMessage
class CustomMessage(BaseMessage):
    def __init__(self, data):
        self.data = data

    @classmethod
    def from_data(cls, data):
        pass

    @abstractmethod
    def get_value(self):
        pass
# Register the message class in config.jsonp
{
  "plugins_path": "./plugins",
  "message_class": "CustomMessage"
}
\end{verbatim}

\subsection{Example: Custom Serializer}\label{subsec:example:-custom-serializer}
To create a custom serializer, implement the required interface and place it in the plugins directory. For example, a custom serializer might look like this:
\begin{verbatim}[language=Python]
from event_bus.plugins import BaseSerializer
class CustomSerializer(BaseSerializer):
    def serialize(self, obj):
        # Custom serialization logic
        pass

    def deserialize(self, data):
        # Custom deserialization logic
        pass
# Register the plugin in config.jsonp
{
  "plugins_path": "./plugins",
  "serializer": "CustomSerializer"
}
\end{verbatim}
